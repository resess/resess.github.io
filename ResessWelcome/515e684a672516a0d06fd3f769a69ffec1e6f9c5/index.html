    Access to Lab's GitHub repo

    • Send a mail to Harsha or Yingying with your GitHub username

    Create project in Lab's GitHub repo

    • Create a repo for your project. Name it {year_project_name} e.g. 2018_android_login_exploration. The project name in the example denotes the year in which project was started and a meaningful name, so the viewer can guess what the project is about from its name.
    • Keep it private.
    • Put details about the project in ReadMe file. This should include a short overview of the project, maintainer of the project, pre-requisites and how one can get started. Check GitHub guidelines for more details.
    • Projects in our lab have predefined categories / tags, such as mobile-energy, mobile-security, microservices, etc. Tag your repo with an appropriate category. A repo can also have multiple tags. Refer to this link on how to tag projects.

    Meeting minutes and research notes

    • Create a notebook in OneNote with the following name: {your name - Degree}.
    • Share this notebook with Prof. Julia Rubin and verify if notebook is accessible in the cloud.
    • Maintain meeting minutes, summary of related work, and research notes in this notebook.
    • Create a section for each topic i.e. meeting minutes, summary of related work, etc. Under each section you can create pages. Create page for each sub-topic like "Meeting-July 10,2018" under "Meeting" section.

    Weekly progress report

    • Please post your weekly progress report by Friday/Saturday on Lab's Google group with following contents:
      • List out all your activities relevant to your research in past one week and briefly describe each activity and results if any.
      • Write one paragraph summary per research paper if you have read any. It should concisely and precisely summarize the main message of the paper and the main takeaway.

    Organize contents of research paper in latex files

    Organize contents of your research paper into the followings latex files(.tex):

    • paper.text
      • This file should contain the title of your paper, the headers, and inclusion of the files that you need for your paper (as below).
    • macro.tex
      • This file should contain all packages, commands, etc. (just copy from the template)
    • abstract.tex
    • intro.tex
    • background.tex
    • approach.tex
    • results.tex
    • related.tex
    • conclusion.tex
    • 00.bib
      • This file should contain the references to research papers, journals, etc.
    • Make file
      • A make file to build the latex package (just copy from the template). Please refer to Research paper template. Run "make" before you commit and make sure you paper compiles every time you commit.

    Note- Please start each sentence in new line in latex files for better reviewability
