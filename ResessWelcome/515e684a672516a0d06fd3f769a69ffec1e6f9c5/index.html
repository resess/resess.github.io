    ReSeSS Lab Procedures

    • Send an e-mail to Prof. Julia Rubin to join lab's Google group.
    • Please make sure you are logged into your gmail account and follow the below steps:
      • Visit ReSeSS calendar.
      • Click on
        button on the right bottom corner. In case you cannot find
        button, refer this image.
      • Mark your availability in the ReSeSS calendar and use the end date for all recurrent events (e.g., mark your classes and set the end date of your classes to the end of the term).
    • Submit a request e-mail to access@ece.ubc.ca with the following information:
      • Your Name: [FULL NAME]
      • Your Student Number : [ID NUMBER]
      • Name of Supervisor: Prof. Julia Rubin
      • Your Roles to Justify Access: Undergrad/Graduate Student
      • Buildings and Rooms to Access: KAIS– 4095
      • Expected Termination Date: [YYYY-MM-DD]
    • To present in an upcoming group meeting, write down your name here under the corresponding date.
    • Create a notebook in OneNote with the following name: {your name - Degree}.
    • Share this notebook with Prof. Julia Rubin at her Gmail account and verify that the notebook is accessible in the cloud.
    • Create at least two sections: Meetings and Related Work.
      • In the Meetings section, create a topic for each meeting and name it with the date of the meeting, e.g., July 10, 2018. Summarize the discussion and action items from the meeting.
      • In the Related Work section, create a short summary (around 2-3 paragraphs) for each paper that you read.
    • You can create other sections relevant to your project, per your convenience.
    • Please post your weekly progress report by Friday/Saturday on lab's Google group with following contents:
      • List out all your activities relevant to your research in past one week and briefly describe each activity and results if any.
      • Write one paragraph summary per research paper if you have read any. It should concisely and precisely summarize the main message of the paper and the main takeaway.

    Computing Infrastructure

    • Complete and print account application form. Both you and your supervisor need to complete and sign it. For your convenience, you can fill in the PDF before printing.
    • Submit the form to the IT Support Team in MacLeod 105, or email it to it@ece.ubc.ca.
    • Check this page for more details.
    • Send an e-mail to Harsha or Yingying with your GitHub username to join lab's GitHub repo
    • Create a repo for your project in lab's GitHub page. Name it {year_project_name} e.g. 2018_android_login_exploration. The project name in the example denotes the year in which project was started and a meaningful name, so the viewer can guess what the project is about from its name.
    • Keep the repo private.
    • Put details about the project in ReadMe file. This should include a short overview of the project, maintainer of the project, pre-requisites and how one can get started. Check GitHub guidelines for more details.
    • Projects in our lab have predefined categories / tags, such as mobile-energy, mobile-security, microservices, etc. Tag your repo with an appropriate category. A repo can also have multiple tags. Refer to this link on how to tag projects.
    • Submit a request e-mail to it@ece.ubc.ca with following information:
      • Your Name: [FULL NAME]
      • Your Student: [ID NUMBER]
      • Name of Supervisor : Prof. Julia Rubin
      • Your ECE mail ID
    • To access Zeus, ssh [your ece mail account]@zeus.ece.ubc.ca
    • If you want to login remotely,
      • login to ece server – ssh [your ece mail account]@ece.ubc.ca
      • login to Zeus machine - ssh [your ece mail account]@zeus.ece.ubc.ca
    • Zeus machine usage:
      • Create a home directory with your username under zeus:/data.
      • Put all your files and install any software in that folder.
      • Put everything that you want to share with the group, including software installations in /data/common, e.g., under /data/common/software/XXX.
    • Two options:
      • GNU Screen
      • Tmux
    • GNU Screen
      • Start screen: screen
      • Detach screen: CTRL-A-D
      • Re-attach latest screen: screen -r
      • Re-attach screen by ID: screen -r [ID]
      • List screen sessions: screen -ls
    • Tmux
      • Start tmux session: tmux new -s [TAG_NAME]
      • Access existing tmux by ID: tmux a [ID]
      • Access existing tmux by tag name: tmux a -t [TAG_NAME]
      • List tmux sessions: tmux ls
      • Kill tmux session: tmux kill-session -t [TAG_NAME]

    Scientific Writing

    • Writting papers in Latex - instructions.
    • Common writing mistakes.
    • Writing good software engineering research papers (by Mary Shaw).
    • Use PowerPoint (not Keynote) for your presentations and format the slides as Standard (4x3), not Widescreen (16x9).
    • Fonts in all presentations should always be at least 20 points.
    • Always include slide numbers.
    • Do not use the equation feature, just symbols.
    • One message per slide, one slide per message. A good slide should not contain more than 30-40 words
    • Do not write in full sentences. Break your message into bullet points.
    • Bullet points should be consistent in structure (verbs vs. nouns, punctuation, etc.).
