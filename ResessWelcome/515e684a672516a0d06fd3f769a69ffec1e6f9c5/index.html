    Scientific Writing

    • Each paper has four main parts of "identification": (1) title, (2) names of ALL authors, (3) publication venue, and (4) year. Always mention all these four parts when describing a paper
    • Conference papers can be published in the main/research track (typically, 10-12 pages), adjunct track (typically, 2-10 pages), or a collocated event, such as a workshop (can be any number of pages). Research track papers are expected to be of the highest quality and level of details. Always mention the track if the paper is not published in the research track.
    • Summarize each and every paper you read, even in a few sentences, in the ReSeSS paper repository. Your future self will thank you, as details will become fuzzy after reading many papers.
    • Writting papers in Latex - instructions.
    • Common writing mistakes.
    • Writing good software engineering research papers (by Mary Shaw).
    • High-level points:
      • Use PowerPoint (not Keynote) for your presentations and format the slides as Widescreen (16x9).
      • Fonts in all presentations should always be at least 18 points.
      • Always include slide numbers.
      • Do not use the equation feature, just symbols.
      • One message per slide, one slide per message.
      • A good slide should not contain more than 30-40 words (20 is the target).
      • Do not write in full sentences. Break your message into bullet points.
      • Bullet points should be consistent in structure (verbs vs. nouns, punctuation, etc.).

    Technical Tips and Tricks

    • Two options:
      • GNU Screen
      • Tmux
    • GNU Screen
      • Start screen: screen
      • Detach screen: CTRL-A-D
      • Re-attach latest screen: screen -r
      • Re-attach screen by ID: screen -r [ID]
      • List screen sessions: screen -ls
    • Tmux
      • Start tmux session: tmux new -s [TAG_NAME]
      • Access existing tmux by ID: tmux a [ID]
      • Access existing tmux by tag name: tmux a -t [TAG_NAME]
      • List tmux sessions: tmux ls
      • Kill tmux session: tmux kill-session -t [TAG_NAME]

    On Zeus

    1. Include the following in your PATH variable, if not there already:
      /emulator
      /platform-tools/
      /cmdline-tools/tools/bin

      E.g., use the following command:
      export PATH=$PATH:sdk_path/emulator/

    2. Check permissions of /dev/kvm. If it does not have crwxrwxrwx+ permissions, give it to the file using the following command:
      sudo chmod 777 /dev/kvm
    3. Create a VNC server and write down its id.
      vncserver (id is after "... desktop is zeus:")
    4. Find the IP address, and write it down.
      ifconfig (The IP address is inet addr for eth0)

    On Your Machine

    1. Install VNC viewer if not done already:
      https://www.realvnc.com/en/connect/download/viewer/
    2. Make sure you are connected to UBC network (ubcprivate or via VPN)
    3. Open VNC viewer, sign up or sign in.
    4. Open the new connection window:
      File->New connection->General
    5. Set the VNC server to be Zeus.
      ip_address:590 + id of VNC server written down in Step 3.
    6. Double click to connect to the viewer.
    7. Input VNC password
    8. You are ready to go. For example, open a terminal window to run any command:
      Right click->Debian->Applications->Terminal Emulators->LXTerminal
    1. Follow the steps in Section "Setting Up a VNC Viewer on Zeus" to set up VNC viewer
    2. Open VNC viewer, connect to Zeus, and open a terminal window
    3. On Zeus, create an emulator and write down its name:
      avdmanager create avd -n test -k "sdk_id"
    4. Create a script, android_emu.sh, with the following code and make the script executable.
      #!/bin/bash
      adb kill-server
      adb start-server
      emulator -avd $1 -port 5554 -gpu guest -writable-system -no-snapshot
    5. Run the script created in Step 4, use the emulator name written down in Step 3 (it is passed to the script as the first parameter). For example,
      ./android_emu.sh test
    6. Wait and you should see the emulator!
