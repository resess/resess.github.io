    Writting Papers in LaTeX

    1. Organize your research paper into the followings latex files (.tex):

    • paper.tex
      • This file should contain the title of your paper, the headers, and inclusion of the files that you need for your paper (as below).
    • macro.tex
      • This file should contain all packages, commands, etc. (just copy from the template).
    • abstract.tex
    • intro.tex
    • background.tex
    • approach.tex
    • results.tex
    • related.tex
    • conclusion.tex
    • 00.bib
      • This file should contain the references to research papers, journals, etc.
      • Bibtex entries of the references should be in this format: last:names:of:all:authors:venue:year.
      • All paper titles should be in title case. The title should be surrounded with "{{}}", to preserve the title capitalization.
      • Conference papers should be referenced as:
        • @InProceedings{Hao:Liu:Nath:Halfond:Govindan:MobiSys:2013,
        author = {Shuai Hao and Bin Liu and Suman Nath and William G.J. Halfond and Ramesh Govindan},
        title = {{PUMA: Programmable UI-Automation for Large-Scale Dynamic Analysis of Mobile Apps}},
        booktitle = {Proc. of the 12th Annual International Conference on Mobile Systems, Applications, and Services (MobiSys)},
        year = {2014},
        pages = {204--217}
        }
      • Journal papers should be referenced as:
        • @article{Li:Zhu:Rubin:Chechik:TSE:2018,
        author={Yi Li and Chenguang Zhu and Julia Rubin and Marsha Chechik},
        title={{Semantic Slicing of Software Version Histories}},
        journal={IEEE Transactions on Software Engineering (TSE)},
        year={2018},
        volume={44},
        number={2},
        pages={182--201}
        }
      • Books should be referenced as:
        • @book{Babbie:book:2015,
        author={Earl Babbie},
        title={{The Practice of Social Research}},
        year={2015},
        publisher={Nelson Education}
        }
      • Technical reports should be referenced as:
        • @techreport{Brun:Holmes:Ernst:Notkin:TechReport:2010,
        author = {Yuriy Brun and Reid Holmes and Michael D. Ernst and David Notkin},
        title = {{Speculative Identification of Merge Conflicts and Non-Conflicts}},
        year = {2010},
        institution = {University of Washington},
        number = {UW-CSE-10-03-01}
        }
      • Tools should be referenced as:
        • @misc{istio,
        author={},
        title = {{Istio: Connect, Secure, Control, and Observe Services}},
        howpublished = {\url{https://istio.io/}},
        year={2019},
        note = {(Last accessed: May 2019)}
        }
      • URLs should be referenced as:
        • @misc{Lewis:Fowler:url:2014,
        author={James Lewis, Martin Fowler},
        title = {{Microservices: A Definition of This New Architectural Term}},
        howpublished = {\url{https://www.martinfowler.com/articles/microservices.html}},
        year={2014},
        note = {(Last accessed: May 2019)}
        }
    • Make file
      • A make file to build the latex package (just copy from the template). Please refer to research paper template. Run "make" before you commit and make sure you paper compiles every time you commit.

    2. All included images should be in .pdf format (not .jpg or .png)

    3. For better readability and version management, start each sentence in a latex file in a new line.

    4. When editing already finalized text, e.g., for a conference/journal major revision or for a submission deadline after Prof. Rubin's pass on the text, mark all you changes (additions and removals), including typo fixes, at word-level.
